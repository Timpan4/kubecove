## Findings considered and rejected

So future audits don't re-report them:

- **Port-allocation race in `start_pod_port_forward_in_registry`**
  (`sessions.rs:493-515`): the OS `TcpListener::bind` is the atomic arbiter
  between the two registry checks; a concurrent explicit-port request fails
  cleanly at bind. Not a bug.
- **`usePodLogStream` unmount/start race**: the `cancelled` flag plus the
  `.then`-side `stopStream` handles every ordering (`usePodLogStream.ts:84-102`).
  Not a bug.
- **Mutex poisoning via `.lock().expect(...)`** (sessions, pod_exec,
  kubeconfig, live_store): poisoning requires a panic while holding short,
  non-panicking critical sections; standard Rust practice. Not worth churn.
- **Client cache clearing all entries at capacity**
  (`client_cache.rs:44-50`): documented design choice ("bounds growth
  without an LRU dependency"); 32 (source, context) pairs is generous for a
  desktop tool. Revisit only with real multi-cluster eviction complaints.
- **ResourceList re-render chain**: the memo chain is correctly structured;
  remaining O(n)-per-keystroke filter cost only matters at ~5k+ resources.
  A `useDeferredValue` on search is a possible future nicety, not a finding.
- **Security advisories (all LOW, no plans)**: `DefaultHasher` for
  kubeconfig source keys (non-secret data, local file); CSP
  `style-src 'unsafe-inline'` (standard for Tauri apps, no remote content);
  plaintext `kubeconfig-sources.json` (contains paths/env-var names, no
  credentials); ANSI passthrough to xterm.js (xterm sanitizes; cluster
  output is trusted per the tool's threat model). The audit found **no
  critical or high security issues** — kubeconfig contents never reach the
  webview, pod exec passes argv arrays, apply validates identity
  server-side and blocks v1 Secrets.
- **3 s session polling** (`refetchInterval: 3_000` ×5 sites): real but the
  maintainer's own `improvements.md` P4 already specifies the better fix
  (push events / shared query); deferred to that backlog rather than
  duplicated here.
- **Zero component-behavior tests** (no testing-library anywhere): real gap,
  but introducing a component-test stack (Tauri IPC mocking + happy-dom +
  testing-library) is an infrastructure decision the owner should make
  deliberately — raise as a discussion, not an executor plan.
- **`pod_exec.rs` over the 800-line hard cap** (838 lines, empty
  `LEGACY_OVERSIZED_FILES` in the hook): tracked in the maintainer's
  `improvements.md` D1; splitting it is prerequisite work for exec lifecycle
  tests (see plan 007's deferred notes).
- **TanStack Router unused** (`@tanstack/react-router` renders a single `/`
  route): already documented as decision A1 in the maintainer's
  `improvements.md` — adopt-or-remove is a product call, not plannable
  without it.
